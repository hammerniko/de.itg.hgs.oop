# Aufgabe
Gegeben ist die Datei `Sequendiagramm.png` in welcher ein Szenario dargestellt ist.

## Klassendiagramm `20min`
Erstellen Sie mithilfe des Sequenzdiagramms ein vollständiges UML-Klassendiagramm.
Beachten Sie dabei:
- Konstruktoren
- Attribute
- Methoden
- Sichtbarkeitskriterien
- Assoziationen
- Rollennamen
- Multiplizitäten


## Quellcode erstellen `20min`
Implementieren Sie die notwendigen Klassen gemäß dem erstellten Klassendiagramm.

## Testen des Szenarios `20min`
Erstellen Sie eine ausführbare Java-Datei um das dargestellte Szenario zu testen.

## Erweiterung der Klassen `30min`

Erweitern Sie die gegebenen Klassen so, dass folgende Forderungen sichergestellt sind:
1. Die Anzahl der erstellten Verzeichnisse soll in der Klasse `Verzeichnis` gespeichert werden.
2. Wird ein Verzeichnis ohne Namen erstellt soll das Verzeichnis `neuerOrder01` heißen. Der Index am Ende soll bei jedem Verzeichnis hochgezählt werden.
3. Der Name eines Verzeichnisses darf nur geändert werden, wenn der Name der entsprechenden `setMethode(..) ` übergeben wurde und existiert. Desweiteren muss der Name midestens ein Zeichen beinhalten.
    >Tipp: Verwenden Sie die Methode `isEmpty()` der Klasse `String`
